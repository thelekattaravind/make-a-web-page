Lesson 1

HTML-hyper text markup language, text content is what you see on the internet and markup is how it looks

How the web works-Your browser on your computer makes HTTP requests which are sent to servers which send back HTML documents to be seen on your browser

Elements-What HTML document are made of

Tags-what is used to manipulate elements, example: < b > < /b >-this is the bold tag it needs both an opening and closing tag around the section of text you want to highlight like THIS but this is not true for all tags example: < br > this produces a line break like the ones you see throughout this HTML document

Inline Tags-the two last discussed tags are called inline tags because they occur in line and are different from "cousin" Block tags

Block tags- are tags that create a block around what ever is effected by the tag making that area separate from the rest of the area

Finally remember computers are stupid they need to be told what to do and that can only happen when you tell them what to do for example if you don't close a tag the computer will make all of the following text bold so remember it is you who needs to write the instructions

Lesson 2

Developer tools- as we discussed in the last lesson there are tags that create boxes around the info you put between the tags. These tools allow you to see the boxes and the properties within the boxes and even the styles that apply to the box.

Structure of HTML- When looking at the bare text of a HTML document you will see a distinct tree-like structure sort of like a genealogy tree where there nested boxes which contain the next generation of family members. The more ideated the box is within the text the deeper nested that box is within its corresponding box.

Text Editors- Meant to make a programmers life easier by providing shortcuts such as automatically providing a closing tag to help the programmer speed along especially considering how much a programmer has to type.

Lesson 3

Avoiding repetition- By not repeating the same process over and over again you can avoid making errors. How so? Simply because if you have to make any changes then you will not have to repeat the same process over and over again by chance missing one fix. Consistency is key because it allow you to define a kind of item and allows you to apply the same process to several objects that are of the same kind.

CSS- Cascading Style Sheets is similar to what was said about consistency CSS allows us to group similar HTML elements under the same class and apply a style to all elements that fall under the same class.
